# 基础知识整理

本文件对应跟踪估计业务域，目的是把“量测、状态、滤波、关联”四个核心概念先整理清楚，再去读 `tracking` 代码。

## 1. 这一层解决什么问题

跟踪估计层关心的不是“某一时刻测到了什么”，而是：

- 当前目标状态的最佳估计是什么
- 下一时刻大概会出现在什么位置
- 这次量测是否属于这条航迹

因此它属于“状态估计链”，而不是“探测链”的简单延续。

## 2. 阅读代码前应先掌握的概念

### 2.1 状态和量测不是一回事

状态通常包含：

- 位置
- 速度
- 有时还包含加速度或机动参数

量测则可能只有位置或方位/距离。滤波的核心任务，就是用动力学模型把不完整、带噪的量测转成更完整的状态估计。

### 2.2 预测与更新

卡尔曼滤波最重要的结构是两步循环：

1. 预测
2. 更新

预测用系统模型把上一时刻状态推到当前时刻，更新用当前量测修正预测误差。

### 2.3 关联门限

有多个量测或多个目标时，必须先解决“哪个量测属于哪条航迹”。

这一层的关键概念包括：

- 创新
- 创新协方差
- 归一化创新平方
- 关联门限

## 3. 与代码的对应关系

- `include/xsf_math/tracking/kalman_filter.hpp`
  提供状态预测、量测更新和协方差演化。
- `include/xsf_math/tracking/track_association.hpp`
  提供量测关联、距离门限和评分逻辑。

## 4. 常见误区

### 4.1 把滤波当成平滑器

滤波不是“把折线抹平”这么简单。它同时承担动力学预测、误差传播和量测融合。

### 4.2 忽略过程噪声

过程噪声不是“给滤波器一点随机扰动”。它表达的是模型本身不完美，例如目标可能机动，而恒速模型无法完全描述这个机动。

### 4.3 只看估计值，不看协方差

协方差决定了后续关联门限、制导可信度和轨迹稳定性。只看状态估计而忽略协方差，跟踪链会失去自我评估能力。
