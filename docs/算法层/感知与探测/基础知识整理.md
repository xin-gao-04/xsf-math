# 基础知识整理

本文件对应感知与探测业务域，目的是把雷达、传播、统计检测和 RCS 之间的关系先讲清楚，再回到 `perception` 相关头文件。

## 1. 这一层解决什么问题

感知与探测层关心的是：

- 目标回波有多强
- 传播损耗后还能剩多少
- 杂波和干扰把门限抬高到什么程度
- 在给定虚警概率下最终还能不能检测到目标

因此这一层不是单个雷达方程，而是一条完整的感知链。

## 2. 阅读代码前应先掌握的概念

### 2.1 雷达方程

雷达方程回答的是“理想接收功率有多大”。它至少依赖：

- 发射功率
- 天线增益
- 波长
- 目标散射截面
- 距离

它给出的只是信号项的量级，还没有加入传播、噪声、杂波和干扰。

### 2.2 传播与方向图

同一个目标在不同传播条件下，探测结果会显著不同。主要原因包括：

- 自由空间路径损耗
- 多径
- 雨衰和大气吸收
- 天线主瓣和旁瓣方向差异

因此 `propagation.hpp` 和 `antenna.hpp` 不是附属模块，而是将理想雷达方程修正到工程环境里的必要部分。

### 2.3 RCS 与起伏模型

RCS 决定了目标“平均会反射多少能量”，但真实目标回波还会在脉冲间起伏。

当前文档和代码主要涉及：

- 非起伏目标近似
- Swerling 系列起伏模型
- Marcum-Q 检测概率计算

这意味着“同样的平均 RCS”并不一定对应“同样的检测性能”。

## 3. 与代码的对应关系

- `include/xsf_math/core/rcs.hpp`
  提供目标散射截面和相关基础模型。
- `include/xsf_math/radar/antenna.hpp`
  提供方向图和增益相关计算。
- `include/xsf_math/radar/propagation.hpp`
  提供路径损耗、衰减和传播修正。
- `include/xsf_math/radar/radar_equation.hpp`
  汇总信号项、噪声项和探测量级估计。
- `include/xsf_math/radar/clutter.hpp`
  提供杂波建模和背景退化项。
- `include/xsf_math/radar/marcum_swerling.hpp`
  提供统计检测和起伏目标检测概率估计。

## 4. 常见误区

### 4.1 只看 SNR，不看门限

检测问题不是单纯比较“信号是不是大于噪声”。还需要门限，而门限又依赖虚警概率、积分方式和目标起伏模型。

### 4.2 把 RCS 当成固定常数

RCS 在姿态、频率和极化变化下都可能变化。工程里若直接把它当成固定量，只能得到某一种条件下的平均近似。

### 4.3 把传播和检测模型拆开理解

传播、杂波、干扰和检测门限是同一条链上的连续环节。分开看有助于理解模块，但使用时必须把它们重新接回到同一条检测链。
