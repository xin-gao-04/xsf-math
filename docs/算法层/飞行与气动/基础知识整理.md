# 基础知识整理

本文件对应飞行与气动业务域，目的是在阅读 `aero` 代码前，先把动压、升阻、过载和能量状态的关系理顺。

## 1. 这一层解决什么问题

飞行与气动层关心的是：

- 当前环境下能产生多大气动力
- 平台还能承受多大过载
- 当前速度、高度和阻力对能量状态有什么影响

因此它是“可执行能力层”，不是动作编排层。

## 2. 阅读代码前应先掌握的概念

### 2.1 动压

动压是气动力的直接来源：

```text
q = 0.5 * rho * V^2
```

它把大气密度和速度统一到一个量上，是升力、阻力和最大过载估计的核心输入。

### 2.2 升力、阻力和升阻比

飞行器能否机动，首先取决于当前可提供的升力。升力增加通常伴随阻力增加，因此大机动会消耗更多能量。

### 2.3 最大过载和比剩余功率

最大过载告诉我们“当前还能拉多狠”，比剩余功率告诉我们“当前还能不能继续爬升或加速”。

这两个量分别对应：

- 瞬时机动能力
- 持续机动能力

## 3. 与代码的对应关系

- `include/xsf_math/core/atmosphere.hpp`
  提供密度、音速、动压和马赫数计算。
- `include/xsf_math/aero/aerodynamics.hpp`
  提供二维气动、升阻计算、过载能力和能量估算。

## 4. 常见误区

### 4.1 把速度当成唯一能力指标

速度高不等于机动能力一定强。高空低密度下，即使速度不低，动压仍可能不足。

### 4.2 把过载限制当成纯结构限制

当前库里不少过载限制来源于气动能力，而不是结构极限。也就是说，不是“危险所以不能做”，而是“做不到”。

### 4.3 忽略气动和行为层的关系

行为层输出的是控制命令，但这些命令是否可执行，最终仍要回到飞行与气动层的动压和过载边界。
