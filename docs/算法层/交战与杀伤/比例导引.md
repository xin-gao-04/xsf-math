# 比例导引与制导计算

> 本文对应 `include/xsf_math/guidance/proportional_nav.hpp`，重点解释制导模块为什么要把“几何计算”和“导引律”分开。

## 模块背景

制导的核心问题不是“飞机朝目标飞过去”这么简单，而是：

- 目标在动
- 目标可能在机动
- 武器机动能力有限
- 越晚修正，末段所需过载越大

比例导引之所以成为主流，是因为它不盯着目标当前位置猛追，而是试图消除视线旋转，让弹目连线逐步稳定下来。

## 1. 当前实现能力

当前制导模块包括：

- `engagement_geometry`
- `proportional_nav`
- `augmented_proportional_nav`
- `pursuit_guidance`
- `accel_limiter`
- 阶段切换条件 `phase_transition`

## 2. 为什么先有 `engagement_geometry`

制导律真正依赖的不是“目标坐标”本身，而是相对几何量：

- 弹目相对位置
- 相对速度
- 斜距
- 闭合速度
- LOS 单位向量
- LOS 角速度
- 预计拦截时间

把这些量先统一算出来的意义是：

- 不同制导律共享同一套输入
- 上层更容易比较不同制导律的效果
- 后面接 `accel_limiter` 或气动约束时也更自然

## 3. 比例导引为什么有效

当前 PN 形式为：

$$
\mathbf{a}_{cmd} = N \cdot (\boldsymbol{\omega}_{LOS} \times \mathbf{V}_{weapon})
$$

其中：

- `N` 是导航比
- `omega` 是 LOS 角速度
- `V_weapon` 是武器速度

这条公式最重要的物理意义是：

- 只要 LOS 还在转，说明当前几何还不是碰撞路径
- LOS 转得越快，说明当前修正不够，需要更大机动
- 叉积保证指令主要作用在改变飞行方向，而不是无意义地改变量值大小

换句话说，比例导引不是“追目标”，而是“消视线转率”。

## 4. 导航比 `N` 的作用

`N` 决定修正有多激进。

- `N` 太小
  修正不够，末段可能来不及拦截
- `N` 太大
  容易过度修正，浪费能量并放大噪声

工程上常用的导航比通常在一个比较保守的区间内，因为制导律不是单独存在的，它还要和：

- 量测噪声
- 目标机动估计误差
- 最大过载限制

一起工作。

## 5. 增强比例导引

APN 在 PN 的基础上额外叠加目标加速度估计。
当前实现中，目标加速度项以负半系数进入：

$$
\mathbf{a}_{cmd} = N \cdot (\boldsymbol{\omega}_{LOS} \times \mathbf{V}_{weapon} - 0.5 \cdot \mathbf{a}_{target})
$$

它解决的是“目标在机动时，普通 PN 总是慢半拍”的问题。

但它也引入了新的要求：

- 需要更稳定的目标加速度估计
- 估计噪声会直接污染指令

所以 APN 更强，但也更依赖跟踪质量。

## 6. 追踪导引为什么还要保留

`pursuit_guidance` 采用更直观的几何方式：

- 让速度方向朝向目标当前位置
- 通过双叉乘构造机动平面的侧向加速度方向

这种方式实现简单，但通常不如 PN 节能。

它存在的意义主要有两个：

- 作为简单稳定的基线导引律
- 适用于一些不强调末段最优性的场景

## 7. 制导律为什么不能脱离能力限制

理想导引律给出的 `a_cmd` 并不一定可执行。

例如：

- 高空动压很低
- 当前速度太小
- 平台 `CLmax` 有限
- 结构允许过载有限

这就是 `accel_limiter` 的意义：

- 先让制导律表达“理想需要”
- 再让限幅器表达“物理可行”

如果把这两件事混在一个公式里，上层就很难区分“制导思路错了”和“平台能力不够”。

## 8. 阶段切换为什么也放在制导相关模块

当前支持的切换变量包括：

- 飞行时间
- 高度
- 速度 / 马赫数
- 动压
- 目标距离
- 地面距离
- 预计拦截时间
- LOS 方位/俯仰

这些变量本质上都和“什么时候该换制导模式”有关。

例如一条典型链路可能是：

1. 中段用较温和的追踪/巡航策略
2. 进入一定距离后切 PN
3. 动压不足或高度变化时重新限幅

所以阶段切换虽然不是完整状态机，但它提供了把多种制导律串起来的判据接口。

## 9. 与其他模块的关系

制导模块最常和下面几个模块一起使用：

- `tracking`
  提供更稳定的目标状态估计
- `aero`
  给出最大可用机动边界
- `lethality`
  在末端把几何结果转换成命中与起爆条件

因此制导模块的真正位置是：

```mermaid

    TRK["🎯 tracking<br/>滤波后目标状态"] --> GEO["📐 engagement_geometry<br/>弹目相对几何"]
    GEO --> GUIDE["🧭 guidance<br/>PN / APN / Pursuit"]
    GUIDE --> AERO["✈️ aero<br/>可用过载约束"]
    AERO --> LETH["💥 lethality<br/>引信 / Pk"]

```

## 10. API 速查

`guidance/proportional_nav.hpp` 全部符号（`namespace xsf_math`）：

| 符号 | 角色 |
|------|------|
| `engagement_geometry` | 弹目相对几何，提供 `range()`、`closing_speed()`、`los_unit()`、`los_rate()`、`time_to_go()` |
| `proportional_nav` | 经典三维 PN，可配置 `N` |
| `augmented_proportional_nav` | APN，叠加 `-0.5·a_target` |
| `pursuit_guidance` | 追踪导引，双叉乘构造侧向方向 |
| `accel_limiter` | 按当前 `q·S·CLmax/(m·g)` 裁切指令 |
| `phase_transition` + `check_phase_transition` | 按飞行时间、高度、马赫、动压、距离、LOS 角等 10 类变量触发阶段切换 |
