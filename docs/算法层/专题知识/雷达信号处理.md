# 雷达信号处理基础

> 本文面向 `radar/` 模块，解释当前库里的雷达算法为什么这样组织，以及一条完整探测链是怎么构成的。

## 新手入门

可以把雷达理解成“主动发出回声询问，再从噪声里找答案”的系统。

真正困难的地方不在于发射，而在于：

1. 目标回波本来就很弱
2. 距离一远，回波按高次幂衰减
3. 地面、海面、天气和干扰都会把分母抬高
4. 最终还要在统计意义上决定“这到底是不是目标”

所以雷达链路不是一个方程，而是一条从发射、传播、散射、接收、判决一路串下来的计算流程。

## 1. 当前覆盖范围

`xsf-math` 中的雷达相关能力主要包括：

- 发射机/接收机参数建模
- 单基地与双基地雷达方程
- 天线方向图
- 自由空间与大气传播损耗
- 杂波与 MTI 改善
- 干扰背景下的 SNR 评估
- 探测概率与作用距离估算

## 2. 探测链的基本逻辑

一条最小探测链可以抽象成：

```text
目标 RCS
  -> 天线增益
  -> 传播损耗
  -> 接收信号功率
  -> 噪声/杂波/干扰
  -> SNR
  -> Pd
```

这里每一步解决的都是不同问题：

- `RCS`
  目标把多少能量“反回来”
- `antenna`
  雷达在当前指向上能提供多大增益
- `propagation`
  信号沿路径损失了多少
- `radar_equation`
  回波到接收机后还剩多少信号功率
- `clutter + ew`
  分母中多了多少杂波和干扰
- `marcum_swerling`
  给定门限和统计模型，最终被判为目标的概率有多高

## 3. 雷达方程

单基地接收功率的核心形式为：

```text
Pr = (Pt * Gt * Gr * lambda^2 * sigma) / ((4*pi)^3 * R^4 * L)
```

这里最重要的不是把公式背下来，而是理解里面每一项的角色：

- `Pt`
  你发射得有多强
- `Gt, Gr`
  你往目标方向看得是否足够“集中”
- `lambda`
  不同频率下传播和散射行为不同
- `sigma`
  目标是否容易被“照亮并反射回来”
- `R^4`
  单基地探测距离最敏感的项，说明距离增长会极快吞噬信号
- `L`
  系统损耗和传播损耗的汇总

当前实现见：

- `include/xsf_math/radar/radar_equation.hpp`

双基地情形则将 `R^4` 换成 `Rt^2 * Rr^2`，因为发射路径和接收路径被分开建模。

## 4. 为什么 SNR 要拆成 `N + C + J`

本库显式区分：

- `N`: 热噪声
- `C`: 杂波
- `J`: 干扰

因此可表达：

```text
SNR = S / (C + J + N)
```

这样拆开的意义是：

- 热噪声是接收机内部背景
- 杂波来自环境反射
- 干扰来自对抗系统主动注入

它们对系统设计的影响完全不同。

例如：

- 想降低 `N`，通常要靠接收机设计和带宽控制
- 想降低 `C`，通常要靠波束、杂波图或 MTI
- 想降低 `J`，则涉及抗干扰和频谱/几何策略

所以把它们混成一个“总噪声常数”会让上层系统失去分析能力。

## 5. 传播模型为什么单独成模块

当前传播侧提供：

- `free_space_loss_db(...)`
- `blake_attenuation`
- `rain_attenuation`
- `lens_loss_db(...)`
- `multipath_model`
- `total_propagation_loss_db(...)`

传播模块单独存在，是因为它解决的是“环境和路径”问题，而不是“雷达本体”问题。

同一部雷达：

- 晴空和暴雨作用距离不同
- 直视与低空掠海多径情况不同
- 单路径与两径反射下的有效信号不同

这也是为什么传播模型特别适合被外部场景或天气系统直接调用。

## 6. 天线与方向图

当前方向图模型包括：

- 各向同性天线
- 余弦幂方向图
- sinc 平方方向图
- 查表方向图

对应头文件：

- `include/xsf_math/radar/antenna.hpp`

天线模块的核心价值不是“提供几个方向图函数”，而是把“离轴损失”显式表达出来。

如果目标不在主瓣里：

- 即使目标本身 RCS 很大
- 即使距离不远
- 实际可用信号也会迅速下降

所以天线是探测链中最典型的“几何与功率耦合模块”。

## 7. 杂波为什么重要

杂波模块提供：

- 地表类型到典型 `gamma` 的映射
- 杂波斑块面积估算
- 杂波功率估算
- MTI 改善因子

对应头文件：

- `include/xsf_math/radar/clutter.hpp`

杂波的本质不是“额外噪声”，而是和目标一样的回波，只不过来自地面/海面等背景。

这意味着：

- 下视雷达比上视更难
- 掠海低空目标探测难度更高
- MTI 或脉冲多普勒的收益主要体现在压低这部分分母

## 8. 从 `SNR` 到 `Pd`

即使 `SNR` 计算出来了，也还不能直接说“探测到了”。

原因是雷达检测本质上是统计判决：

- 门限太低，虚警多
- 门限太高，漏检多

`marcum_swerling` 模块做的事情，就是在给定：

- 目标起伏模型
- 积累脉冲数
- `Pfa`
- 当前 `SNR`

之后，求出最终的 `Pd`

这一步把“物理功率链”变成了“检测结果链”。

## 9. 与测试/示例的对应关系

- `examples/radar_detection_example.cpp`
- `tests/test_radar.cpp`

这两个入口覆盖了当前雷达链的主要使用方式：

- 先算传播和回波
- 再算 `SNR`
- 最后映射到探测概率和作用距离
