# 验证体系与离线报告

本文档描述当前仓库新增的“直观效果验证 + 数据验证”落地方式。

## 1. 目标

验证体系分两层：

- 直观效果验证
  用场景级示例生成时间序列、关键事件和结论页，让使用者直接看到行为链是否按预期推进。
- 数据验证
  用 `ctest` 和文件驱动样例确保核心公式、插值和控制器输出持续回归。

## 2. 当前验证载体

### 2.1 示例级结构化产物

以下示例已支持 `--strict` 和 `--output-dir`：

- `examples/radar_detection_example.cpp`
- `examples/missile_engagement_example.cpp`
- `examples/orbital_example.cpp`
- `examples/behavior_chain_example.cpp`

当传入 `--output-dir <dir>` 时，每个示例会在 `<dir>/<case_id>/` 下输出：

- `run_summary.json`
- `metrics.json`
- `timeseries.csv`
- `events.jsonl`

### 2.2 回归级验证

- `ctest` 继续承担快速数值回归。
- 示例 smoke 也已纳入 `ctest`，用于覆盖场景链路。

## 3. 批量运行方式

在仓库根目录执行：

```bash
python3 tools/validation/run_cases.py --build-dir build --report
```

默认会：

1. 执行 `cmake --build build -j4`
2. 运行四个结构化示例，并以 `--strict` 模式收集产物
3. 运行一次 `ctest --output-on-failure`
4. 生成离线报告

默认输出目录：

```text
build/validation/
```

可选参数：

```bash
python3 tools/validation/run_cases.py --build-dir build --output-dir build/validation_ci --report
python3 tools/validation/run_cases.py --build-dir build --skip-build --report
python3 tools/validation/run_cases.py --build-dir build --skip-ctest
```

## 4. 报告产物

报告生成脚本：

```bash
python3 tools/validation/build_report.py --input-dir build/validation
```

输出：

- `build/validation/report/report.html`
- `build/validation/report/report.md`

报告内容包括：

- 每个案例的通过/失败结论
- 目的、预期行为、实际结论
- 输入与关键指标表
- 时间序列图
- 事件时间线
- 原始 artifact 链接

## 5. 当前判定逻辑

### 雷达探测样例

- SNR 随距离单调下降
- Sw0 的 `Pd < 0.5` 距离与计算探测距离接近

### 导弹交战样例

- 成功进入 terminal
- 引信触发
- 脱靶量不超过引信半径
- `Pk` 足够高

### 轨道样例

- LEO/GEO 周期保持在合理参考范围
- LOS 结果与当前几何配置一致

### 行为链样例

- 至少确认一条航迹
- 成功发射
- 进入 burst，而不是 break-off

## 6. 设计边界

- 报告当前是离线 HTML/Markdown，不依赖前端工程。
- 图表由 Python 标准库生成的静态 SVG 组成，不依赖第三方绘图库。
- 结构化产物目前优先覆盖示例链路；单元测试仍以断言回归为主。

## 7. 推荐使用流程

1. 先跑 `ctest`，确认数值基线干净。
2. 再跑 `tools/validation/run_cases.py --report`。
3. 先看 `report.html` 的摘要页和失败项。
4. 再沿着失败案例进入 `timeseries.csv`、`events.jsonl` 和 `console.log`。
