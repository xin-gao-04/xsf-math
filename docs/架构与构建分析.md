# xsf-math 架构与构建分析

## 1. 仓库定位

`xsf-math` 当前不是完整仿真框架，而是从 `xsf-core` 中抽离出的可复用能力仓库。

它的定位更接近：

- 头文件级算法库
- 轻量行为控制库
- 上层仿真、武器、传感器模块的底座能力集合

当前仓库不承载场景管理、实体生命周期、事件系统或扩展注册框架，重点只保留可独立复用的计算逻辑。

## 2. 目录结构

当前源码树可以概括为四部分：

1. `include/xsf_math`
   算法层公开头文件，负责数学模型和领域计算。
2. `include/xsf_behavior`
   行为层公开头文件，负责原子飞行动作控制器。
3. `examples`
   最小功能示例，用于串联多个模块验证使用方式。
4. `tests`
   轻量单元/集成测试，可作为回归入口。
5. `docs`
   当前项目的结构化文档，其中算法层文档已收拢到 `docs/算法层/`，行为层文档收拢到 `docs/行为层/`。

## 3. 构建方式

顶层构建入口是：

```text
CMakeLists.txt
```

其关键特征如下：

- 项目为 `C++17`
- `xsf_math` 被定义为 `INTERFACE` 目标
- `include/` 被作为唯一公开头目录导出
- `examples/` 和 `tests/` 通过 CMake 选项控制是否构建

这意味着库本身没有独立编译产物，真正被编译的是示例和测试。

## 4. 当前构建链特点

### 4.1 头文件库

`add_library(xsf_math INTERFACE)` 表明：

- 没有 `.cpp` 编译单元作为库产物
- 接入方通过链接接口目标继承 `include` 路径
- 绝大多数能力都内联在头文件中

### 4.2 示例与测试承担“可编译验证”

由于是头文件库，示例和测试实际上承担了两个角色：

- 演示 API 的典型调用方式
- 验证头文件在真实编译单元中是否可用

这也是当前仓库最有效的回归面。

### 4.3 UTF-8 编译选项

仓库当前启用了：

```cmake
if(MSVC)
    add_compile_options(/utf-8)
endif()
```

原因是仓库已经引入中文注释和文档字符串，MSVC 若不显式使用 UTF-8，容易在当前代码页下产生解析问题。

## 5. 模块结构观察

从聚合头 `include/xsf_math/xsf_math.hpp` 可以看出当前仓库现在包含两层能力：

- 算法层：`core`、`radar`、`tracking`、`guidance`、`aero`、`ew`、`lethality`、`orbital`
- 行为层：`xsf_behavior`

其中行为层建立在算法层之上，但仍然保持“只输出控制意图、不接管状态推进”的边界，适合继续作为可裁剪库来维护。

## 6. 示例与测试布局

### 示例

- `examples/radar_detection_example.cpp`
  关注雷达方程、Pd 和电子战效应。
- `examples/missile_engagement_example.cpp`
  串联制导、气动、引信和 Pk 评估。
- `examples/orbital_example.cpp`
  展示轨道根数、传播和可见性计算。

### 测试

- `tests/test_core.cpp`
  覆盖向量、矩阵、坐标变换、大气、插值和卡尔曼滤波。
- `tests/test_radar.cpp`
  覆盖雷达方程、Marcum-Swerling、天线、传播、杂波和 RCS。
- `tests/test_guidance.cpp`
  覆盖制导、电子战、气动、引信、Pk 与轨道相关能力。
- `tests/test_flight.cpp`
  覆盖平飞、转弯、航点跟踪、进近和拉平等行为控制能力。

## 7. 与 xsf-core 的关系

当前仓库可以理解为 `xsf-core` 的“两层裁剪子集”：

- 算法层：保留可独立成立的数学模型和基础物理工具
- 行为层：保留可脱离运行时、可由外部框架托管的原子飞行动作
- 去除了框架级运行时
- 去除了扩展发现/注册机制
- 去除了复杂的产品化交付链

这也决定了文档应围绕“算法能力 + 行为工作流 + 集成方式”来组织，而不是围绕“大型 SDK 维护流程”来组织。

## 8. 后续维护方向

- 继续保持模块边界清晰，避免重新耦合回框架语义。
- 若后续文档继续扩展，优先补充每个头文件的 API 级说明和输入/输出约束。
- 若后续准备对外发布，可再增加安装导出、版本策略和变更日志文档。
