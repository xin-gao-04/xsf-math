# 原子控制器总览

本文档描述当前 `xsf-behavior` 中各原子动作控制器的职责边界、典型输入和输出意图。

当前这一层不是重新发明一套飞行业务逻辑，而是尽量把 `xsf-core` 中 `GuidanceComputer / GuidanceProgram`
可独立抽取的控制规律拆成轻量接口：

- 纵向控制优先对齐 `AltitudeGuidance` 和 `FlightPathAngleGuidance`
- 横向控制优先对齐 `xsf-core` 的“单步航向变化 -> 侧向加速度”求解方式
- 参数限制优先对齐 `xsf-core` 相位默认值，例如 `25g` 最大指令加速度、`70 deg` 最大俯仰角、`0` 表示爬升/下降率不限

因此，这一层的姿态输出本质上是对 `xsf-core` 加速度指令的适配投影，而不是新的主控制律。

## 公共输入输出

所有控制器共享以下基础结构：

- `flight_kinematic_state`
  输入当前高度、航向、俯仰、滚转、真空速、垂向速度、动压等状态。
- `flight_control_limits`
  输入平台限制，例如最大俯仰、最大滚转、最大航向率、最小速度和最小动压。
  其中重力偏置、横向偏置、最大爬升率、最大下降率语义直接沿用 `xsf-core`。
- `flight_control_command`
  输出俯仰、滚转、航向率、垂向速度、等效法向加速度折算 g 值，以及 `valid / saturated / status`。
  同时保留 `commanded_lateral_accel_mps2 / commanded_vertical_accel_mps2`，便于外部框架直接消费更接近 `xsf-core` 的指令量。

## 动作列表

### `pull_up_controller`

用途：
按 `xsf-core::AltitudeGuidance` 的方式，把高度差先换成所需垂向速度，再换成飞行路径角和纵向指令加速度。

典型输出：

- 正俯仰
- 正垂向速度
- 负向 `ECS Z` 指令加速度（表示向上拉升）

### `coordinated_turn_controller`

用途：
按 `xsf-core` 的单步航向变化逻辑，先求所需侧向加速度，再投影成滚转和航向率。

典型输出：

- 左/右滚转
- 同号航向率
- 协调转弯附加过载

### `descent_controller`

用途：
按 `xsf-core::AltitudeGuidance` 的方式，根据目标高度直接形成下降路径。

典型输出：

- 负俯仰
- 负垂向速度

### `level_hold_controller`

用途：
仍然使用高度制导逻辑，但目标高度通常就是当前保持高度，因此更接近平飞保持语义。

典型输出：

- 小幅俯仰修正
- 小幅垂向速度修正

### `waypoint_track_controller`

用途：
根据二维航点位置生成水平面转向命令。

典型输出：

- 朝航点方向的滚转和航向率
- 到达半径内返回 `invalid_target`

### `approach_glideslope_controller`

用途：
横向按阈值点求航向变化，纵向按下滑道投影出的目标高度做 `AltitudeGuidance`。

典型输出：

- 朝阈值点方向的滚转和航向率
- 负垂向速度
- 按距离阈值点投影得到的期望下滑道俯仰

### `heading_hold_controller`

用途：
把当前航向拉回一个给定航向，适合进入稳定航向段后的细修正。

典型输出：

- 朝目标航向回中的滚转和航向率

### `flare_controller`

用途：
在接地前低高度阶段，把目标下沉率换成目标飞行路径角，再走 `FlightPathAngleGuidance`。

典型输出：

- 抬头俯仰
- 更温和的下降率

## 当前适用方式

这些控制器适合被外部仿真框架按时间步调用：

1. 外部框架维护平台状态
2. 每一步选择当前动作
3. 调用控制器得到命令
4. 将命令映射到底层姿态或动力学更新

当前仓库不直接提供动作调度器或阶段机。
