# 原子控制器总览

本文档描述当前 `xsf-behavior` 中各原子动作控制器的职责边界、典型输入和输出意图。

## 公共输入输出

所有控制器共享以下基础结构：

- `flight_kinematic_state`
  输入当前高度、航向、俯仰、滚转、真空速、垂向速度、动压等状态。
- `flight_control_limits`
  输入平台限制，例如最大俯仰、最大滚转、最大航向率、最小速度和最小动压。
- `flight_control_command`
  输出俯仰、滚转、航向率、垂向速度、法向过载，以及 `valid / saturated / status`。

## 动作列表

### `pull_up_controller`

用途：
将高度误差和目标爬升角转换成抬头和爬升指令。

典型输出：

- 正俯仰
- 正垂向速度
- 大于等于 `1g` 的法向过载

### `coordinated_turn_controller`

用途：
根据目标航向输出协调转弯所需的滚转和航向率。

典型输出：

- 左/右滚转
- 同号航向率
- 协调转弯附加过载

### `descent_controller`

用途：
根据目标高度和下滑角输出下降动作。

典型输出：

- 负俯仰
- 负垂向速度

### `level_hold_controller`

用途：
在目标高度附近抑制大动作，把平台拉回稳定平飞。

典型输出：

- 小幅俯仰修正
- 小幅垂向速度修正

### `waypoint_track_controller`

用途：
根据二维航点位置生成水平面转向命令。

典型输出：

- 朝航点方向的滚转和航向率
- 到达半径内返回 `invalid_target`

### `approach_glideslope_controller`

用途：
同时做横向对准和纵向下滑道保持，适合基础进近。

典型输出：

- 朝阈值点方向的滚转和航向率
- 负垂向速度
- 按距离阈值点投影得到的期望下滑道俯仰

### `heading_hold_controller`

用途：
把当前航向拉回一个给定航向，适合进入稳定航向段后的细修正。

典型输出：

- 朝目标航向回中的滚转和航向率

### `flare_controller`

用途：
在接地前低高度阶段抬头并降低下沉率，形成最后一段拉平控制意图。

典型输出：

- 抬头俯仰
- 更温和的下降率

## 当前适用方式

这些控制器适合被外部仿真框架按时间步调用：

1. 外部框架维护平台状态
2. 每一步选择当前动作
3. 调用控制器得到命令
4. 将命令映射到底层姿态或动力学更新

当前仓库不直接提供动作调度器或阶段机。
