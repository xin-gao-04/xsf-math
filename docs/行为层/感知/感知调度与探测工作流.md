# 感知调度与探测工作流

本文档描述从传感器调度到探测判决的完整数据流。

## 1. 调度器工作流

### 1.1 两种模式

```mermaid
flowchart LR
    subgraph 搜索["🔍 搜索模式"]
        S1["轮询 search_list"] --> S2["按帧时间均匀分配驻留"]
    end
    subgraph 跟踪["🎯 跟踪模式"]
        T1["检查 next_visit_time"] --> T2["到期即服务"]
    end

    style S1 fill:#ffedd5
    style T1 fill:#e1f5e1
```

### 1.2 跟踪优先策略

```mermaid
flowchart TD
    TIME["当前仿真时间"] --> FIND{"查找所有跟踪请求<br/>next_visit_time <= 当前时间"}
    FIND -->|找到| EARLIEST["选最早到期的请求"]
    FIND -->|无到期| CHECK_SEARCH{"搜索是否轮到？"}
    CHECK_SEARCH -->|是| NEXT_SEARCH["取下一个 search_slot"]
    CHECK_SEARCH -->|否| IDLE["本周期无事"]
    EARLIEST --> UPDATE_TRACK["更新该请求的<br/>next_visit_time += revisit_interval"]
    NEXT_SEARCH --> UPDATE_SEARCH["更新 next_search_visit<br/>+= search_chance_interval"]

    style EARLIEST fill:#e1f5e1
    style IDLE fill:#ffedd5
```

### 1.3 关键参数

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `search_frame_time_s` | 2.0 s | 搜索一圈的总时间 |
| `track_revisit_time_s` | 0.5 s | 跟踪默认再访问间隔 |
| `dwell_time_s` | 0.010 s | 单次驻留时间 |
| `max_track_requests` | 16 | 最大并发跟踪请求数 |

### 1.4 典型工作流

```mermaid
flowchart LR
    START(["初始化"]) --> ADD_SEARCH["填充 search_list"]
    ADD_SEARCH --> LOOP["每帧调用 select"]
    LOOP -->|track mode| HANDLE_TRACK["输出跟踪指令<br/>外部照射后反馈探测结果"]
    LOOP -->|search mode| HANDLE_SEARCH["输出搜索指令<br/>外部照射后反馈探测结果"]
    HANDLE_TRACK -->|发现目标| NEW_TRACK["add_track_request"]
    HANDLE_SEARCH -->|未发现| LOOP
    NEW_TRACK --> LOOP
    HANDLE_TRACK -->|丢失航迹| DROP["drop_track_request"]
    DROP --> LOOP
```

## 2. 探测控制器工作流

### 2.1 输入准备

外部框架在调用 `detection_controller::evaluate` 前，需要组装：

```cpp
detection_inputs in;
in.geometry.range_m = ...;           // 到目标的斜距
in.geometry.target_rcs_m2 = ...;     // 目标 RCS
in.geometry.tx_antenna_gain_db = ...;// 指向目标的发射增益
in.geometry.rx_antenna_gain_db = ...;// 指向目标的接收增益
in.geometry.elevation_rad = ...;     // 指向目标的俯仰角
in.clutter_power_w = ...;            // 杂波功率（可选）
in.jamming_power_w = ...;            // 干扰功率（可选）
in.ew_effect = ...;                  // EW 降级乘子（可选）
in.js_ratio_linear = ...;            // 干信比（可选）
```

### 2.2 判决流程

```mermaid
flowchart TD
    IN["detection_inputs"] --> STEP1["1️⃣ 单基地雷达方程<br/>monostatic_radar_equation"]
    STEP1 --> STEP2["2️⃣ 并联噪声+杂波+干扰<br/>snr_with_interference"]
    STEP2 --> STEP3["3️⃣ 叠加 EW 降级<br/>ew_effect.degrade_snr"]
    STEP3 --> STEP4["4️⃣ Marcum-Swerling 映射<br/>compute_pd"]
    STEP4 --> CHOICE{"stochastic?"}
    CHOICE -->|true| RAND["5️⃣ 伯努利抽样<br/>rand() < Pd"]
    CHOICE -->|false| THRES["5️⃣ 确定性门限<br/>Pd >= threshold"]
    RAND --> OUT["detection_decision"]
    THRES --> OUT

    style STEP4 fill:#dbeafe
    style OUT fill:#e1f5e1
```

### 2.3 输出解读

```cpp
struct detection_decision {
    bool   detected;        // 本次驻留是否判为有目标
    double snr_linear;      // 有效 SNR（线性值）
    double snr_db;          // 有效 SNR（dB）
    double pd;              // 本次的探测概率
    double false_alarm;     // 虚警概率（来自检测器参数）
    double sample_draw;     // 抽样的随机值（随机模式）或门限（确定性模式）
};
```

## 3. 搜索与跟踪的协同

### 3.1 搜索发现新目标

```mermaid
sequenceDiagram
    participant F as 外部框架
    participant S as sensor_scheduler
    participant D as detection_controller
    participant T as track_manager

    F->>S: select(sim_time)
    S-->>F: cmd {mode=search, target_index=3}
    F->>D: evaluate(inputs_for_target_3)
    D-->>F: decision {detected=true}
    F->>T: start_tentative_track(det, sim_time)
    T-->>F: track_id = 7
    F->>S: add_track_request(7, 3, sim_time)
```

### 3.2 跟踪维持与回退

```mermaid
sequenceDiagram
    participant F as 外部框架
    participant S as sensor_scheduler
    participant D as detection_controller
    participant T as track_manager

    loop 每 revisit_interval
        F->>S: select(sim_time)
        S-->>F: cmd {mode=track, request_id=7}
        F->>D: evaluate(inputs_for_track_7)
        D-->>F: decision {detected=true/false}
        F->>T: update(detections, sim_time)
        T-->>F: result {confirmed_tracks/dropped/...}
    end
```

## 4. 当前边界

当前工作流尚未覆盖：

- **多传感器协同调度**：多部雷达的时序协调
- **自适应 revisit 间隔**：根据航迹质量动态调整跟踪频率
- **确认前跟踪（TWS）**：边搜索边跟踪的混合模式
- **多普勒处理**：速度门、MTI 改善因子的实时计算
- **红外/光电探测**：只有雷达链路
