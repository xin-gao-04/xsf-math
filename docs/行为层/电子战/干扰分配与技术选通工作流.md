### 1.3 选择逻辑

```mermaid
flowchart TD
    SSJ["SSJ_J/S"] & SOJ["SOJ_J/S"] --> COMPARE{"SOJ >= SSJ?"}
    COMPARE -->|是| MODE_SOJ["mode = stand_off<br/>js = SOJ_J/S"]
    COMPARE -->|否| MODE_SSJ["mode = self_screening<br/>js = SSJ_J/S<br/>burnthrough = 计算烧穿距离"]
    MODE_SOJ & MODE_SSJ --> THRES{"js >= js_threshold?"}

    THRES -->|否| INEFFECTIVE["effective = false"]

    style EFFECTIVE fill:#e1f5e1
    style INEFFECTIVE fill:#ffedd5
```

### 1.4 典型参数

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `jammer.erp_w` | 1000 | 有效辐射功率 |
| `jammer.bandwidth_hz` | 1e9 | 干扰带宽 |
| `radar_tx_power_w` | 100000 | 雷达发射功率 |
| `radar_tx_gain_linear` | 1000 | 发射增益（线性） |
| `target_rcs_m2` | 5.0 | 目标 RCS |
| `js_threshold_linear` | 1.0 | 有效压制门限（J/S = 1 = 0 dB） |

## 2. 技术选通工作流

### 2.1 技术库注册

```mermaid
flowchart LR
    REG["add_technique(ew_technique)"] --> LIB["存入 library<br/>key = technique_id"]

    style LIB fill:#dbeafe
```

每条技术包含：
- `technique_id`：唯一标识
- `mitigation_class_id`：对方反制匹配 ID
- `target_function`：针对的系统功能（search/track/guidance/communication）
- `effect`：EW 降级乘子（jamming_power_gain, noise_multiplier, blanking_factor）

### 2.2 选通与去选通

```mermaid
flowchart TD
    SELECT["select(id, function)"] --> CHECK1{"技术在库中?"}
    CHECK1 -->|否| FAIL1["返回 false"]
    CHECK1 -->|是| CHECK2{"技术适用该功能?"}
    CHECK2 -->|否| FAIL2["返回 false"]
    CHECK2 -->|是| ADD["加入 selected_ids"]
    ADD --> OK["返回 true"]

    DESELECT["deselect(id)"] --> REMOVE["从 selected_ids 移除"]

    style OK fill:#e1f5e1
    style FAIL1 fill:#ffedd5
```

### 2.3 效果合成

```mermaid
flowchart LR
    SELECTED["selected_ids"] --> FOR["遍历每个选通技术"] --> FILTER{"适用目标功能?"}
    FILTER -->|是| APPLY["乘子相乘<br/>blanking 取最小"]
    FILTER -->|否| SKIP["跳过"]
    APPLY --> OUT["combined_effect"]

    style OUT fill:#e1f5e1
```

合成规则：
```cpp
// 干扰和噪声：乘性叠加
out.jamming_power_gain *= e.jamming_power_gain;
out.noise_multiplier   *= e.noise_multiplier;

// blanking：取最小（最保守）
out.blanking_factor = min(out.blanking_factor, e.blanking_factor);
```

## 3. 与探测/交战链的协同

### 3.1 完整降级链路

```mermaid
sequenceDiagram
    participant F as 外部框架
    participant JA as jam_assignment
    participant ET as ew_technique
    participant DET as detection_controller
    participant TRK as track_manager
    participant FUZE as fuze_controller

    F->>JA: select(inputs)
    JA-->>F: cmd {mode, js_ratio, effective}

    F->>ET: combined_effect_for(search)
    ET-->>F: effect {noise_mult, jam_gain, blanking}

    F->>DET: evaluate(inputs with ew_effect)
    DET-->>F: decision {detected, snr, pd}

    Note over TRK: 探测率降低 → 航迹质量下降

    F->>FUZE: evaluate(fuze_inputs with track_error)
    FUZE-->>F: decision {estimated_pk}
```

### 3.2 多函数选通示例

```cpp
// 对搜索雷达使用技术 A
ew_ctrl.select("tech_A", ew_system_function::search);

// 对跟踪雷达使用技术 B
ew_ctrl.select("tech_B", ew_system_function::track);

// 分别获取效果
auto search_effect = ew_ctrl.combined_effect_for(ew_system_function::search);
auto track_effect  = ew_ctrl.combined_effect_for(ew_system_function::track);
```

## 4. 当前边界

当前电子战工作流尚未覆盖：

- **欺骗式干扰技术**：RGPO、VGPO、角度欺骗（算法层无模型）
- **雷达告警接收机（RWR）**：截获、识别、威胁排序
- **电子防护（EP）**：频率捷变、PRF 抖动、旁瓣对消
- **红外对抗（IRCM）**：调制干扰、DIRCM
- **多干扰机协同**：多部干扰机的时频协调
- **低截获概率（LPI）对抗**：波形设计、截获因子
