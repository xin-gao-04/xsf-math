# 原子控制器总览

本文档描述当前 `xsf-behavior` 中各原子动作控制器的职责边界、典型输入和输出意图。

原生制导程序层用于保留程序级执行顺序。
本文档描述的是其上的外部框架适配动作层。

这一层将可独立使用的控制规律拆成轻量接口：

- 纵向控制基于高度和航迹角制导
- 横向控制基于单步航向变化和侧向加速度转换
- 参数限制采用统一的姿态、过载和升降率约束

姿态输出由加速度指令投影得到，不承担程序编排职责。

## 公共输入输出

所有控制器共享以下基础结构：

- `flight_kinematic_state`
  输入当前高度、航向、俯仰、滚转、真空速、垂向速度、动压等状态。
- `flight_control_limits`
  输入平台限制，例如最大俯仰、最大滚转、最大航向率、最小速度和最小动压。
  其中还包括重力偏置、横向偏置、最大爬升率和最大下降率。
- `flight_control_command`
  输出俯仰、滚转、航向率、垂向速度、等效法向加速度折算 g 值，以及 `valid / saturated / status`。
  同时保留 `commanded_lateral_accel_mps2 / commanded_vertical_accel_mps2`。

## 动作列表

### `pull_up_controller`

用途：
把高度差先换成所需垂向速度，再换成飞行路径角和纵向指令加速度。

典型输出：

- 正俯仰
- 正垂向速度
- 负向 `ECS Z` 指令加速度（表示向上拉升）

### `coordinated_turn_controller`

用途：
先求所需侧向加速度，再投影成滚转和航向率。

典型输出：

- 左/右滚转
- 同号航向率
- 协调转弯附加过载

### `descent_controller`

用途：
根据目标高度直接形成下降路径。

典型输出：

- 负俯仰
- 负垂向速度

### `level_hold_controller`

用途：
仍然使用高度制导逻辑，但目标高度通常就是当前保持高度，因此更接近平飞保持语义。

典型输出：

- 小幅俯仰修正
- 小幅垂向速度修正

### `waypoint_track_controller`

用途：
根据二维航点位置生成水平面转向命令。

典型输出：

- 朝航点方向的滚转和航向率
- 到达半径内返回 `invalid_target`

### `approach_glideslope_controller`

用途：
横向按阈值点求航向变化，纵向按下滑道投影出的目标高度做 `AltitudeGuidance`。

典型输出：

- 朝阈值点方向的滚转和航向率
- 负垂向速度
- 按距离阈值点投影得到的期望下滑道俯仰

### `heading_hold_controller`

用途：
把当前航向拉回一个给定航向，适合进入稳定航向段后的细修正。

典型输出：

- 朝目标航向回中的滚转和航向率

### `flare_controller`

用途：
在接地前低高度阶段，把目标下沉率换成目标飞行路径角，再走 `FlightPathAngleGuidance`。

典型输出：

- 抬头俯仰
- 更温和的下降率

## 与 xsf-core 的关系

原子控制器层在 xsf-core 中没有直接对应的类。它是基于 xsf-core 相同的核心算法构建的外部框架适配层：

- `altitude_guidance_command` → 内部复用 xsf-core `AltitudeGuidance` 的完整算法（sqrt(g*Δh) 所需垂向速度 → 圆弧法向加速度 → cos(fpa)*g 重力偏置）
- `flight_path_angle_guidance_command` → 内部复用 xsf-core `FlightPathAngleGuidance` 的完整算法（pitchDelta/dt*speed → 减去重力 ECS 分量）
- `heading_change_command` → 使用标准协调转弯公式（heading_change/dt*speed → atan2(lateral_accel, g) 滚转角）

这些控制器在 xsf-core 中属于 mover 或外部行为管理器的职责范围，当前库将其抽取为独立可调用的原子单元。

## 关键实现细节

### 纵向控制中的重力偏置

所有纵向高度控制器在计算法向加速度后，都会叠加 `cos(fpa) * gee_bias * g`。这个偏置的作用是在目标高度差为零时维持平飞，否则平台会因为没有抵消重力分量而持续下沉。`gee_bias_g` 默认为 1.0，与 xsf-core 行为一致。

### 低能量保护

所有控制器共享 `flight_controls_available` 门限检查。当速度低于 `min_speed_mps`（默认 40 m/s）或动压低于 `min_dynamic_pressure_pa`（默认 500 Pa）时，控制器拒绝输出有效命令并报告 `speed_limited` 或 `dynamic_pressure_limited` 状态。

### 速率限制

所有姿态输出都经过 `apply_rate_limit`，确保单个时间步内的姿态变化不超过平台可执行的角速率上限。

## 当前适用方式

这些控制器适合被外部仿真框架按时间步调用：

1. 外部框架维护平台状态
2. 每一步选择当前动作
3. 调用控制器得到命令
4. 将命令映射到底层姿态或动力学更新

当前仓库不直接提供动作调度器或阶段机。
