# 原生制导程序层

本文档描述 `include/xsf_behavior/flight/guidance_programs.hpp` 这一层的定位。

这一层承载程序级制导流程。目标是把可独立复用的程序链拆成可由外部框架直接调用的原子单元。

## 1. 当前已迁入的程序

- `legacy_guidance_program`
- `altitude_guidance_program`
- `intercept_guidance_program`
- `legacy_flight_path_angle_program`
- `gravity_bias_program`
- `gravity_turn_program`
- `attitude_guidance_program`
- `flight_path_angle_guidance_program`
- `orbit_insertion_program`

## 2. 这一层和原子控制器层的区别

原生制导程序层保留的是程序级执行顺序：

- LOS 导引先算 `pursuit / PN / APN`
- 再叠加 `angle_offset`
- 再按需要做时间常数平滑
- 再处理 `flight_path_angle / altitude`
- 最后叠加 `gravity_bias / lateral_bias`

原子控制器层保留的是对外更好接入的动作接口：

- 拉升
- 转弯
- 定高
- 航点跟踪
- 进近
- 拉平

原生制导程序层用于程序链驱动场景。
原子控制器层用于动作托管场景。

## 3. 当前公共数据

程序层统一依赖：

- `guidance_program_state`
- `guidance_phase_options`
- `guidance_program_commands`
- `guidance_program_result`

其中 `guidance_phase_options` 收纳了关键相位参数，例如：

- `pn_gain`
- `vp_gain`
- `cos_los_offset`
- `gee_bias`
- `lateral_gee_bias`
- `max_gee_cmd_mps2`
- `max_pitch_angle_rad`
- `max_ascent_rate_mps`
- `max_descent_rate_mps`

## 4. 程序链结构

### `legacy_guidance_program`

执行链如下：

1. 根据 LOS 与 `cos_los_offset` 判断使用 `pursuit` 还是 `PN/APN`
2. 叠加 `angle_offset`
3. 按 `time_constant` 平滑横纵向指令
4. 若存在 `commanded_flight_path_angle`，覆盖纵向指令
5. 若存在 `commanded_altitude`，覆盖纵向指令
6. 若纵向程序未覆盖，则叠加 `gravity_bias`
7. 最后叠加 `lateral_bias`

### `intercept_guidance_program`

封装拦截程序中的主要切换参数：

- `pursuit_nav_gain`
- `pro_nav_gain`
- `cos_switch_angle`
- `pn_method`

### `altitude_guidance_program`

纵向高度程序的核心计算步骤：

- 高度差先换算为所需垂向速度
- 再换算为飞行路径角
- 再换算为完成该角度变化的纵向加速度
- 同时支持 `AGL/MSL` 语义

### `legacy_flight_path_angle_program`

旧式飞行路径角程序的核心行为：

- 根据目标飞行路径角输出纵向加速度
- 使用统一完成判据

### `gravity_bias_program`

单独暴露重力偏置计算，便于在不运行完整 legacy program 时复用。

### `gravity_turn_program`

保持“纵向不主动附加额外俯仰控制、主要依赖重力转弯”的程序语义。

### `attitude_guidance_program`

按姿态角或姿态角速率生成三轴角速率命令，支持机体系角和惯性系目标角两种输入方式。

### `flight_path_angle_guidance_program`

提供完整飞行路径角程序。

- 支持固定俯仰角速率和时间常数两类过程
- 支持抬头段和压头段的不同收敛过程
- 输出纵向加速度和俯仰角速率

### `orbit_insertion_program`

提供轨道插入程序。

- 依据轨迹倾角、速度比例和阈值区间切换粗调与细调
- 通过重力偏置修正向圆轨道状态收敛
- 在满足条件后给出完成状态

## 5. 与 xsf-core 的对齐验证

所有制导程序均已与 xsf-core 的 `XsfGuidanceProgram.cpp` 逐项对比验证：

| 程序 | xsf-core 对应 | 状态 |
|------|---------------|------|
| `legacy_guidance_program` | `LegacyProgram::Execute` | ✅ 一致 |
| `intercept_guidance_program` | `InterceptProgram::Execute` | ✅ 一致 |
| `altitude_guidance_program` | `AltitudeProgram::Execute` | ✅ 一致 |
| `legacy_flight_path_angle_program` | `LegacyFlightPathAngleProgram::Execute` | ✅ 一致 |
| `gravity_bias_program` | `GravityBiasProgram::Execute` | ✅ 一致 |
| `gravity_turn_program` | `GravityTurnProgram::Execute` | ✅ 一致 |
| `attitude_guidance_program` | `AttitudeProgram::Execute` | ✅ 一致 |
| `flight_path_angle_guidance_program` | `FlightPathAngleProgram::Execute` | ✅ 一致 |
| `orbit_insertion_program` | `OrbitInsertionProgram::Execute` | ✅ 一致 |

核心辅助函数对比：

| 函数 | xsf-core 对应 | 状态 |
|------|---------------|------|
| `pro_nav_guidance` | `ProNavGuidance` | ✅ omega/叉乘/APN -0.5系数一致 |
| `pursuit_guidance` | `PursuitGuidance` | ✅ 双叉乘法一致 |
| `altitude_guidance` | `AltitudeGuidance` | ✅ sqrt(g*Δh)/圆弧加速度/cos(fpa)*g偏置一致 |
| `flight_path_angle_guidance` | `FlightPathAngleGuidance` | ✅ pitchDelta/dt*speed 减重力ECS分量一致 |
| `apply_gravity_bias` | `ApplyGravityBias` | ⚠️ 使用平地 {0,0,g} 替代 mu/r² |
| `apply_lateral_bias` | `ApplyLateralBias` | ✅ 一致 |
| `angle_offset_guidance` | `AngleOffsetGuidance` | ✅ 一致 |

已知差异：

- `apply_gravity_bias`：xsf-core 使用 `ComputeGravitationalAcceleration`（基于位置计算 mu/r²，方向指向地心），xsf_behavior 使用简化的平地引力 `{0, 0, g}`。对于近地面场景完全等效。`orbit_insertion_program` 在内部自行计算 `mu_earth / radius²`，因此高空场景不受影响。

## 6. 当前未覆盖部分

这一层尚未覆盖完整的程序族，仍包括以下缺口：

- 更完整的程序配置输入优先级链
- 与 mover 命令接口更贴近的速度/推力/最小离地高度约束
- 更细的程序完成判定和阶段切换条件

## 7. 接入方式

外部框架存在对象模型和状态推进时，可按以下顺序接入：

1. 把平台状态适配成 `guidance_program_state`
2. 把阶段参数适配成 `guidance_phase_options`
3. 选择一个 guidance program 执行
4. 直接消费 `accel_cmd_ecs / angle_rate_cmd_rad_s`
5. 再由外部框架把这些命令映射到底层 mover 或动力学模型
