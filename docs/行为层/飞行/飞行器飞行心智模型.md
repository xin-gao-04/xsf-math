# 飞行器飞行心智模型

> 本文为行为层建立思维框架。不解释单个控制器的接口，而是回答"当你要让一个飞行器在仿真中正确飞起来、飞到某处、安全降落时，到底需要考虑哪些事情"。

## 0. 为什么需要这份心智模型

行为层的代码只有几百行，但它背后隐含了大量"为什么这么做"的判断。如果不先建立心智模型，就会遇到以下困境：

- 看到 `cos(fpa) * g` 不知道为什么要加
- 看到 `sqrt(g * delta_h)` 不知道为什么不是线性
- 看到控制器拒绝输出不知道是 bug 还是设计
- 不知道什么时候该切换动作、什么时候不该

这份文档从"飞行器飞行时要面对哪些物理现实"出发，帮你建立一张完整的思维地图。

## 1. 飞行器飞行的四个核心约束

任何飞行器在任何时刻都同时受到四类约束，缺一不可：

### 1.1 能量约束

飞行器是靠动能和势能在天上待着的。

- 速度代表动能
- 高度代表势能
- 推力增加总能量，阻力消耗总能量
- 做机动（拉过载）会加速消耗能量

这意味着：
- 不是想拉多大过载就能拉——过载越大，诱导阻力以平方增长，速度下降越快
- 爬升是在用动能换势能，下降是反过来
- 导弹末段如果能量耗尽，即使制导律给出了正确方向也追不上目标

在代码中：`fuel_model` 和 `specific_excess_power` 跟踪的是能量状态。`dynamic_pressure` 是能量在气动层的直接体现。

### 1.2 气动约束

飞行器靠空气产生控制力。没有气动力，控制指令就是废纸。

气动约束的核心链条：

```text
高度 → 密度 → 动压 → 可用升力 → 可用过载
```

关键量：

- **动压** `q = 0.5 * rho * V²`：气动力的来源。动压为零 = 无法机动
- **最大升力系数 CLmax**：平台几何决定的气动上限
- **最大可用过载** `n_max = q * S * CLmax / (m * g)`：能量和气动共同决定的硬上限

这就是为什么行为层所有控制器都先检查 `flight_controls_available`——如果动压不够，不是"输出小一点"而是"直接拒绝输出"。

### 1.3 几何约束

飞行器有姿态、有惯性、有速度方向。它不能瞬间转向。

- **航迹角**：速度方向与水平面的夹角。不是俯仰角——俯仰角是机体朝向，航迹角是实际飞行方向
- **航向**：速度在水平面的投影方向
- **滚转**：绕速度轴的旋转，用于改变升力方向

改变航迹角需要法向力，改变航向需要侧向力。这些力来自气动，但受限于上面的能量和气动约束。

这就是为什么每次改变高度或航向，都要经过"目标变化量 → 所需角速率 → 所需加速度 → 是否超限"这条链。

### 1.4 时间约束

仿真是离散的。控制器每个时间步只能发出一次命令。

- 时间步内能改变的姿态量 = 角速率 × dt
- 如果目标要求的变化超过单步能力，必须限幅
- 限幅后实际动态可能与理想轨迹偏离

这就是 `apply_rate_limit` 存在的原因：它不是"安全保护"，而是"物理现实"——飞行器不可能在一个时间步内完成 90 度转弯。

## 2. 纵向飞行：高度控制的思维链

纵向控制回答的问题是：怎么从当前高度到达目标高度。

### 2.1 思维链

```text
Δh（高度差）
  → 所需垂向速度 Vv = sqrt(g * |Δh|)
    → 所需航迹角 γ = asin(Vv / V)
      → 航迹角变化 Δγ = γ_target - γ_current
        → 法向加速度 a = V² / R（圆弧假设）
          → 叠加重力偏置 a += cos(γ) * g
            → 限幅到可用过载范围
              → 输出 ECS Z 加速度指令
```

### 2.2 为什么是 sqrt(g*Δh) 而不是线性

如果用线性比例 `Vv = k * Δh`，远处飞太慢，近处飞太快。

`sqrt(g * Δh)` 来源于等减速假设：以 1g 持续减速从某垂向速度停下来恰好走过 Δh 的距离。这给出了"远了快飞、近了自然减速"的平滑收敛行为。

如果 Δh 很大，Vv 可能超过当前飞行速度，此时 `asin(Vv/V)` 会超界，必须限幅到最大俯仰角——这就是"饱和"的来源。

### 2.3 为什么需要重力偏置

当 Δh = 0 且 γ = 0 时，如果不施加任何法向力，飞行器会因为重力持续获得向下的加速度分量。

`cos(γ) * g` 恰好是重力在垂直于速度方向的分量。加上这个偏置后，"维持当前航迹角"不再需要额外指令——这就是"平飞保持"的物理基础。

### 2.4 航迹角 vs 俯仰角

这是最容易混淆的概念：

- **航迹角 γ**：速度矢量与水平面的夹角。这决定了飞行器实际是在升还是在降
- **俯仰角 θ**：机体纵轴与水平面的夹角。这是"机头朝哪"

两者之差就是**迎角 α = θ - γ**。迎角是产生升力的来源。

在行为层中，纵向制导的核心变量是航迹角，不是俯仰角。输出的 `commanded_pitch_rad` 更接近航迹角目标值，而非真正的机体俯仰角。这与 xsf-core 的设计一致。

## 3. 横向飞行：航向控制的思维链

横向控制回答的问题是：怎么把飞行器的水平航向从当前值转到目标值。

### 3.1 思维链

```text
Δψ（航向差，归一化到 [-π, π]）
  → 所需航向角速率 ω = Δψ / dt
    → 所需侧向加速度 a_lat = ω * V_ground
      → 叠加横向偏置
        → 限幅到可用加速度范围
          → 滚转角 φ = atan2(a_lat, g)
            → 限幅到最大滚转角
              → 速率限幅到单步可达滚转
                → 输出滚转、航向率、侧向加速度
```

### 3.2 协调转弯的物理模型

飞行器通过倾斜（滚转）让升力矢量的水平分量提供向心力。

在协调转弯中：

```text
L * sin(φ) = m * V² / R     （向心力）
L * cos(φ) = m * g            （支持重力）
→ tan(φ) = V² / (R * g) = a_lat / g
→ φ = atan2(a_lat, g)
```

所以 1g 侧向加速度 → 45° 滚转，2g → ~63°。

过载为 `1 / cos(φ)`：60° 滚转 = 2g 过载。这就是为什么转弯本身也受过载约束限制。

### 3.3 航向归一化

航向差必须归一化到 `[-π, π]`，否则从 350° 转到 10° 会被算成 -340° 而不是 +20°。`normalize_angle_pm_pi` 解决的就是这个问题。

这在实际代码中极易出错。任何涉及航向差的计算，都必须先经过归一化。

## 4. 纵横耦合：为什么不能分开看

实际飞行中，纵向和横向不是独立的：

- 转弯时升力的一部分用于维持高度，一部分用于改变方向
- 滚转角越大，可用于维持高度的升力分量越小
- 如果同时大幅转弯和大幅爬升，两者争夺过载余量

行为层通过 `merge_commands` 处理这个问题：取纵向和横向中更大的过载需求作为合并输出。

进近控制器（`approach_glideslope_controller`）是最典型的纵横耦合场景：横向对准跑道方向，纵向跟踪下滑道，两个指令同时生成并合并。

## 5. 制导和飞控的分界

行为层处在制导和飞控之间。理解它的定位需要看清三层结构：

### 第一层：制导——"该往哪飞"

制导算法（PN、追踪、航迹角程序等）解决的是方向问题：

- 给出理想的加速度方向和大小
- 不关心平台当前能不能做到
- 不关心执行器怎么实现

### 第二层：行为/动作控制——"怎么飞过去"

行为层解决的是"把意图转成可执行的命令"：

- 把高度目标转成俯仰命令
- 把航向目标转成滚转命令
- 把加速度指令裁到气动能力范围内
- 把姿态变化裁到单步速率范围内
- 判断当前能量是否足以执行

### 第三层：飞控/动力学——"具体怎么动"

飞控层解决的是舵面、推力矢量等具体执行器的控制问题：

- 把俯仰命令转成升降舵偏转角
- 把滚转命令转成副翼偏转差动
- 涉及真正的六自由度动力学

当前仓库只覆盖第一层和第二层。第三层由外部仿真框架负责。

## 6. 低能量状态：飞行中最危险的区域

当飞行器速度过低或高度过低时，进入所谓"低能量状态"。这时：

- 动压不足，气动控制力急剧下降
- 制导给出的指令无法被执行
- 强行执行可能导致失速

行为层用两个门限保护这种情况：

- `min_speed_mps`：速度门限
- `min_dynamic_pressure_pa`：动压门限

低于门限时控制器直接返回 `valid = false`。

这不是保守的安全余量，而是物理上不可能：没有动压就没有气动力，发出的控制指令没有任何效果。

## 7. 飞行阶段与动作切换

一次完整飞行可以分为几个典型阶段：

### 7.1 离地/发射段

- 速度从零开始增加
- 动压从零开始建立
- 控制器可能处于"拒绝输出"状态直到动压足够
- 导弹场景中可能用 `gravity_turn_program` 或 `flight_path_angle_guidance_program`

### 7.2 爬升/巡航段

- 能量充足，气动控制力充分
- 纵向用 `pull_up_controller` 或 `level_hold_controller`
- 横向用 `waypoint_track_controller` 或 `heading_hold_controller`
- 这是最"正常"的飞行状态

### 7.3 拦截/交战段

- 制导律主导（PN/APN/追踪）
- 加速度指令可能接近过载上限
- 能量消耗快速增加
- 末段可能面临能量耗尽

### 7.4 下降/进近段

- 高度在减少，速度可能需要控制
- `descent_controller` 或 `approach_glideslope_controller`
- 纵横耦合最明显：同时对准和下降

### 7.5 着陆/拉平段

- 高度极低，容错空间极小
- `flare_controller` 接管：把下沉率线性收敛到接地速率
- 越接近地面，目标下沉率越小
- 这是对控制精度要求最高的阶段

### 7.6 轨道段

- 离开大气层，气动控制完全失效
- 控制力来自推力矢量
- `orbit_insertion_program` 通过重力偏置修正飞行路径角
- 速度达到轨道速度后宣布入轨

每个阶段的切换由外部框架负责。行为层只回答"在当前阶段应该输出什么命令"。

## 8. 导弹 vs 飞机：同一框架，不同侧重

同一套行为层框架适用于导弹和飞机，但侧重不同：

|  | 导弹 | 飞机 |
|--|------|------|
| 典型制导 | PN/APN 拦截 | 航点跟踪、定高定航向 |
| 能量管理 | 一次性，不可补充 | 可持续，推力可调 |
| 过载范围 | 20-40g 常见 | 2-9g 常见 |
| 飞行时间 | 秒到分钟 | 分钟到小时 |
| 低能量风险 | 末段能量耗尽 | 低速失速 |
| 进近/着陆 | 不需要 | 核心阶段 |
| 关键程序 | `legacy_guidance_program`、`intercept_guidance_program` | 原子控制器链 |

理解这个差异有助于选择正确的控制器和合理的参数范围。

## 10. 常见误解

### "俯仰角就是航迹角"

不是。俯仰角是机头朝向，航迹角是实际飞行方向。两者之差是迎角。在低攻角飞行中近似相等，但在大机动或大迎角状态下差异显著。

### "动压不够时小一点力也能控制"

不能。气动力 = 动压 × 面积 × 系数。动压为零则力为零。低动压时应该等待速度建立，而不是发出弱控制指令。

### "过载限制只是安全余量"

不是。过载限制反映的是当前动压和质量下，气动面物理上能产生的最大法向力。超过这个值不是"危险"，而是"做不到"。

### "转弯不消耗能量"

消耗。转弯需要增大升力系数，升力系数增大导致诱导阻力以平方增长。大角度持续转弯会显著掉速。

### "重力偏置是可选的"

不可选。在近地面飞行中，不加重力偏置的平飞保持会持续丢高度。这不是"优化"，是"不加就不对"。

### "纵向和横向可以独立设计"

不能完全独立。它们共享同一个过载预算。大滚转会减少可用于维持高度的升力分量。合并命令时必须考虑这个耦合。
